eros开发中遇到的坑(持续更新中)

官网地址

最近公司要做APP,用到了eros,对于这个嘛!!!我表示不赞同(有啥用),因为之前没接触过,所以还是很期待的,说说最近遇到的几个问题

问题全部是我目前第一次接触eros所遇到的,记录下来,有的找到解决办法了,有的没有,没有解决的,后续解决了补上

直接来正题

  • eros只支持px单位,可以忽略不写px,盒模型的 box-sizing 默认为 border-box ,布局模型基于 CSS Flexbox

    这里有写

  • 常用标签div text image scroller list slider

    1. div里的内容不会滚动,超出屏幕部分不会显示

    2. text 里面放文本内容,不能直接放在别的容器里

    3. list 相当于ul cell相当于li ,里面的内容可以滚动

    4. <scroller> 是一个竖直的，可以容纳多个排成一列的子组件的滚动器。如果子组件的总高度高于其本身，那么所有的子组件都可滚动。

      注意： <scroller> 可以当作根元素或者嵌套元素使用。此组件的滚动方向是垂直方向的形式。

    5. 不允许相同方向的 <list> 或者 <scroller> 互相嵌套，换句话说就是嵌套的 <list>/<scroller> 必须是不同的方向。

      • 问题一: 现在需要一个页面里有两个滚动区域,<list> 或者 <scroller>不能同时一个方向
      • 解决办法: 利用弹层可以实现..目前还没找到合适的弹层,ui框架提供的弹层,都不能适应需求
      • 后续找到了补上………………….

  • audio问题,weex只提供了video标签,目前正在寻找解决途径!!!!后续补上…………………….

  • 大部分的ui框架都不支持,已知 AMUI BUI-Weex Weex-ui weex-flymeui

  • 目前我大部分再用 Weex-ui ,发现的问题以下几个

    1. wxc-tab-page常用于 Tab 切换页面，目前支持icon、text、iconFont形式的顶栏

      • 问题一: 滑动效果下的border样式,只支持固定宽度,所以需要tabbar的字数一致,如果tabbar的字数不一致,那……..
      • 解决办法: 原生手写一个………………..

    2. wxc-ep-slider用于丰富 slide，包括商品橱窗，图片列表等

      • 问题一: 当然是个小问题了,但也是个小坑,就是如果你不去设置scal呢e 这个属性的话,Android是没啥问题的,iOS问题就大了,这个问题让我纠结了,当时完全不知道错在哪里了,还以为是iOS这个不兼容

      • 解决办法:scale 一定要设置上,1 为水平, 以下是其他的缩放

  • 运行weex debug 时可能会出现错误,我第一次安装的时候觉得太慢了就把他取消了,运行weex debug 也可以跑起来,但是不能进行调试,原因是少安装了一个包(Chromium) ,
    看cmd 有个错误Error: AssertionError [ERR_ASSERTION]: Chromium revision is not downloaded. Run “npm install”会提示你npm install 不要管他 ,更新你的weex-debugger,weex update weex-debugger
    更新完后 运行weex debug 会再次安装weex-debugger ,然后重要的是还有一个(Chromium) 的安装 很慢 ,但一定要等他安装完 ,才可以 然后就OK了
    附上 解决地址 :https://github.com/weexteam/weex-toolkit/issues/408
    这个还是挺不小心的,主要是也能跑起来weex debug ,所以刚开始没往那方面想,被这个小坑了下
